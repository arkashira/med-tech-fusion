# Breakeven Analysis
## Cost per Active User (CAU)
We estimate the following costs per active user:

| Component | Cost per Active User (USD) |
| --- | --- |
| Compute (AWS EC2) | $0.025/hour (avg. 8 hours/day) = $1.20/day = $43.20/month |
| Storage (AWS S3) | $0.023/GB-month (avg. 100 GB/user) = $2.30/month |
| Bandwidth (AWS CloudFront) | $0.085/GB (avg. 100 GB/user) = $8.50/month |
| Total | $54.00/month |

## Pricing Tiers

| Tier | Price (USD/mo) | Features |
| --- | --- | --- |
| Basic | $99 | 1 project, 10 users, 100 GB storage, 100 GB bandwidth |

## Customer Acquisition Cost (CAC) Range
Based on industry benchmarks, we estimate the CAC range for med-tech-fusion to be between $200 and $500 per user.

## Lifetime Value (LTV) Estimate
We estimate the LTV for med-tech-fusion to be around $1,500 per user, assuming an average subscription duration of 12 months and a 20% monthly retention rate.

## Break-even Users Count
To break even, we need to acquire a certain number of users. Assuming a CAC of $300 and an LTV of $1,500, we can calculate the break-even point as follows:

Break-even users count = (CAC x 12) / (LTV - CAC)
= ($300 x 12) / ($1,500 - $300)
= 3.6 users

## Path to $10K MRR
To reach $10,000 in monthly recurring revenue (MRR), we need to acquire a certain number of users. Assuming an average revenue per user (ARPU) of $333 (based on the pricing tiers), we can calculate the required number of users as follows:

Required users = $10,000 / $333
= 30 users

To reach this goal, we can focus on acquiring users through the Pro tier, which has an ARPU of $499. We estimate that we need to acquire around 20 Pro users to reach the break-even point and an additional 10 Pro users to reach $10,000 in MRR.

| Tier | Number of Users | ARPU (USD) | MRR (USD) |
| --- | --- | --- | --- |
| Pro | 20 | $499 | $9,980 |
| Pro | 10 | $499 | $4,990 |
